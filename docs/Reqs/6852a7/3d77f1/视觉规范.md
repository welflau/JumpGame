# 视觉规范 — 搭建游戏基础框架和Canvas渲染系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 整体风格
**像素完美复古风格**：采用 8-bit / 16-bit 游戏美学，所有图形资源禁用抗锯齿和平滑处理，保持清晰的像素边缘。色彩方案使用深色背景（深蓝/深紫色调）搭配高对比度的霓虹色强调元素（红粉/青绿），营造赛博朋克与复古街机的混合氛围。UI 文字统一使用像素字体并添加硬边阴影增强可读性，所有动画和过渡保持帧数感（避免平滑缓动），加载进度条采用逐格填充而非渐变过渡，确保整体视觉与 60 FPS 固定时间步长的技术实现相呼应。

## 组件视觉规范

# 组件视觉规格

## Canvas 容器
- **尺寸**: 800x600 像素（固定）
- **背景色**: #1a1a2e（深蓝灰）
- **渲染模式**: 像素完美（image-rendering: pixelated / crisp-edges）
- **边框**: 无
- **居中方式**: CSS margin auto，页面背景 #0a0a14

## LOADING 状态界面
- **背景**: 纯色填充 #16213e
- **标题文字**: "LOADING" 居中显示
  - 字体: 32px 像素字体（Press Start 2P 或等宽字体）
  - 颜色: #eef4ed（浅灰白）
  - 位置: Canvas 垂直居中偏上 40px
  - 文字阴影: 2px 2px #000000
- **进度条**:
  - 位置: 标题下方 60px
  - 外框: 400x24px，边框 2px #0f3460，背景 #0f3460
  - 填充: 从左到右填充 #e94560（红粉色），宽度 = 400px * (progress / 100)
  - 进度文字: "XX%" 显示在进度条下方 16px，字体 16px，颜色 #eef4ed

## PLAYING 状态界面
- **背景层**: 纯色 #0f3460 或占位渐变（深蓝到浅蓝）
- **游戏对象层**: 预留透明层，供后续绘制角色/敌人/平台
- **UI 层**: 左上角预留 HUD 区域（16px padding）
  - 占位文字: "Game Ready" 或 "FPS: 60"
  - 字体: 14px 像素字体，颜色 #eef4ed，文字阴影 2px 2px #000000

## VICTORY / DEFEAT 状态界面
- **背景**: 半透明遮罩 rgba(0, 0, 0, 0.7) 覆盖游戏画面
- **状态文字**: "VICTORY" / "DEFEAT" 居中显示
  - 字体: 48px 像素字体
  - 颜色: VICTORY #00ff00（绿色），DEFEAT #ff0000（红色）
  - 文字阴影: 3px 3px #000000
- **提示文字**: "Press R to Restart" 显示在状态文字下方 80px
  - 字体: 16px，颜色 #eef4ed

## Debug 信息层（可选）
- **位置**: 右上角 16px padding
- **内容**: FPS 计数器、Delta Time、当前状态
- **字体**: 12px 等宽字体，颜色 #00ff00（绿色）
- **背景**: 半透明黑色 rgba(0, 0, 0, 0.5)，padding 8px

## 精灵图技术要求

- **所有精灵**: 透明背景 PNG，颜色深度 32-bit RGBA
- **像素对齐**: 所有坐标和尺寸必须是整数像素，禁止亚像素渲染
- **调色板**: 限制在 16-32 色以内，保持复古风格
- **导出设置**: 禁用抗锯齿，使用最近邻插值
- **命名规范**: `{entity}_{action}_{frame}.png` 或 `{entity}_{action}_sheet.png`
- **组织结构**: `assets/sprites/{entity}/` 目录分类存放
