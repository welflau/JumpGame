# 视觉规范 — 实现摄像机跟随系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 整体风格
本系统为纯逻辑功能层，无独立视觉风格。摄像机跟随采用「隐形设计」原则：玩家不应感知到摄像机的存在，只体验到流畅的视角移动。调试模式下使用极简线框风格（红/绿/黄单色线条 + 等宽字体），确保调试信息清晰可读但不干扰游戏画面。所有游戏对象保持原有视觉风格，摄像机系统仅负责坐标转换，不改变任何渲染样式。

# 摄像机跟随系统视觉规格

## 1. 摄像机视口（Camera Viewport）
- **尺寸**：全屏填充（width = canvas.width, height = canvas.height）
- **边界**：无可见边框（摄像机本身不渲染）
- **调试模式边框**：红色 1px 实线（#FF0000），仅开发环境显示

## 2. 跟随目标指示器（Follow Target Indicator - 调试用）
- **形状**：十字准星
- **颜色**：黄色（#FFFF00）
- **尺寸**：8x8px
- **位置**：屏幕宽度 1/3 处（玩家理想跟随位置）
- **显示条件**：仅调试模式

## 3. 安全区域标识（Safe Zone - 调试用）
- **区域**：屏幕左侧 1/3 宽度矩形
- **填充色**：半透明绿色（rgba(0, 255, 0, 0.1)）
- **边框**：绿色 1px 虚线
- **显示条件**：仅调试模式

## 4. HUD 调试信息面板
- **背景色**：半透明黑色（rgba(0, 0, 0, 0.5)）
- **文字颜色**：白色（#FFFFFF）
- **字体**：Courier New 12px（等宽字体）
- **内边距**：8px
- **位置**：屏幕左上角
- **内容**：
  - Camera X: {value}
  - Target X: {value}
  - Player X: {value}
  - Lerp Active: {true/false}
- **显示条件**：仅调试模式

## 5. 边界警告指示器（Boundary Warning - 调试用）
- **左边界**：屏幕左侧红色竖线 2px（camera.x = 0 时显示）
- **右边界**：屏幕右侧红色竖线 2px（camera.x = levelWidth - screenWidth 时显示）
- **显示条件**：仅调试模式

## 6. 状态颜色规范
- **正常跟随**：无特殊视觉反馈
- **贴左边界**：调试模式显示左侧红线
- **贴右边界**：调试模式显示右侧红线
- **插值停止**：调试信息 "Lerp Active: false" 显示为灰色

## 7. 过渡动画规范
- **摄像机移动**：线性插值（lerp factor = 0.1），无缓动曲线
- **停止阈值**：误差 ≤ 1px 时停止插值
- **帧率要求**：60 FPS 下平滑无卡顿

## 精灵图规格
# 摄像机跟随系统精灵图规格

## 无需新增精灵图
本工单为纯逻辑实现，不涉及任何新的精灵图资产。

## 现有精灵图适配要求
所有现有游戏对象的精灵图需满足以下技术规格，以确保与摄像机系统兼容：

### 1. 坐标系统要求
- **世界坐标**：所有游戏对象必须存储世界坐标（worldX, worldY）
- **渲染坐标**：render() 方法必须接收 camera 参数并调用 camera.worldToScreen() 转换坐标
- **锚点**：精灵图锚点统一为左上角（0, 0），确保坐标转换一致性

### 2. 渲染方法签名
所有游戏对象的 render() 方法必须遵循以下签名：
```javascript
render(ctx, camera) {
  const screenPos = camera.worldToScreen(this.x, this.y);
  ctx.drawImage(this.sprite, screenPos.x, screenPos.y, this.width, this.height);
}
```

### 3. 精灵图尺寸约束
- **最大尺寸**：单个精灵图不超过屏幕宽度（避免跨视口渲染问题）
- **最小尺寸**：不小于 8x8px（确保在移动设备上可见）
- **推荐尺寸**：16x16 / 32x32 / 64x64（2 的幂次方便于缩放）

### 4. 动画精灵图要求
- **帧间距**：统一使用固定帧宽（如 32px）
- **帧数**：建议 4-8 帧（平衡流畅度和性能）
- **帧率**：12-24 FPS（与摄像机 60 FPS 更新频率解耦）

### 5. 背景图层适配
- **视差层**：背景图需实现视差滚动（移动速度 = 摄像机速度 × 视差系数）
- **无限滚动**：背景图宽度 ≥ 屏幕宽度 × 2（支持无缝循环）
- **固定层**：HUD 元素不使用 worldToScreen 转换（固定在屏幕坐标）

## 调试模式精灵图（开发专用）

### 1. 摄像机边界指示器
- **类型**：程序生成图形（非精灵图）
- **尺寸**：2px × 屏幕高度（竖线）
- **颜色**：红色 #FF0000
- **动画**：无

### 2. 跟随目标十字准星
- **类型**：程序生成图形（非精灵图）
- **尺寸**：8x8px
- **颜色**：黄色 #FFFF00
- **动画**：可选闪烁效果（1Hz 频率）

### 3. 安全区域遮罩
- **类型**：程序生成矩形（非精灵图）
- **尺寸**：屏幕宽度 / 3 × 屏幕高度
- **颜色**：半透明绿色 rgba(0, 255, 0, 0.1)
- **动画**：无

## 性能优化建议
- **视口裁剪**：仅渲染屏幕可见区域内的精灵图（worldX 在 camera.x - margin 到 camera.x + screenWidth + margin 范围内）
- **精灵图缓存**：使用离屏 canvas 缓存静态精灵图，减少重复绘制
- **LOD（细节层次）**：距离摄像机较远的对象可使用低分辨率精灵图（可选优化）
