# 视觉规范 — 实现摄像机跟随系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 整体风格
Camera system is **invisible infrastructure** with no production UI. Debug overlays use high-contrast neon colors (magenta/green/cyan) on semi-transparent black backgrounds for developer visibility. All visual feedback comes from smooth world object translation at 60 FPS. Debug mode uses monospace fonts and technical readouts styled like developer consoles.

## 组件视觉规范

# Camera System Visual Specification

## Debug Overlay (Development Only)

### Camera Bounds Indicator
- **Type**: Stroke rectangle
- **Color**: Magenta (#FF00FF)
- **Line Width**: 2px
- **Position**: Matches camera viewport boundaries
- **Visibility**: Toggle with F3 key (debug mode)

### Player Safe Zone Guide
- **Type**: Vertical line
- **Color**: Green (#00FF00, 50% opacity)
- **Line Width**: 1px
- **Position**: Screen width × 1/3 from left edge
- **Label**: "Player Target Zone" in 12px monospace white text with black shadow

### Camera Info Panel
- **Position**: Top-left corner, 8px padding
- **Background**: Semi-transparent black (rgba(0,0,0,0.7))
- **Text Color**: White (#FFFFFF)
- **Font**: 12px monospace
- **Content**:
  - Camera X: {value}px
  - Target X: {value}px
  - Player X: {value}px
  - Lerp Active: {true/false}
- **Line Height**: 16px

### Level Boundary Markers
- **Left Boundary**: Red dashed line at world x=0
- **Right Boundary**: Red dashed line at world x=levelWidth
- **Line Style**: 2px dashed, #FF0000
- **Visibility**: Only in debug mode

## Visual Feedback (Production)

### Smooth Transition Indicator
- **No visible UI element** (camera movement itself is the feedback)
- **Behavior**: All world objects translate smoothly via lerp
- **Performance**: 60 FPS target, no visible stuttering

### Edge Clamping Behavior
- **Visual**: Camera stops moving when reaching level boundaries
- **Player continues moving** within visible screen area
- **No UI indicator needed** (natural game feel)

## Coordinate System Visualization (Debug)

### World Coordinates
- **Grid Lines**: Optional 100px grid overlay in cyan (#00FFFF, 20% opacity)
- **Origin Marker**: Small cross at (0,0) in world space

### Screen Coordinates
- **Canvas Border**: 1px solid white outline on canvas element
- **Center Crosshair**: Optional red crosshair at screen center (debug only)
## 精灵图规格
# Camera System Sprite Specification

## Core System
**No sprites required** — Camera is a logical viewport system with no visual representation in production builds.

## Debug Sprites (Development Only)

### Debug Cursor Sprite
- **ID**: `debug_camera_cursor`
- **Size**: 16x16px
- **Format**: Single frame PNG
- **Design**: Crosshair with 4px center dot, 1px lines extending to edges
- **Color**: Yellow (#FFFF00) with 1px black outline
- **Usage**: Marks camera target position in debug mode

### Camera Boundary Icon
- **ID**: `debug_camera_frame`
- **Size**: 32x32px
- **Format**: Single frame PNG
- **Design**: Stylized camera viewfinder corners (4 L-shaped brackets)
- **Color**: Magenta (#FF00FF)
- **Usage**: Decorative icon for debug panel header

## Animation Specs
**No animations required** — Camera movement uses programmatic lerp interpolation, not sprite animation.

## Particle Effects
**Not applicable** — Camera system has no particle effects.

## UI Sprites

### Debug Toggle Button
- **ID**: `btn_debug_camera`
- **Size**: 24x24px
- **States**: 2 frames (off/on)
- **Design**: Camera icon with red slash (off) / green checkmark (on)
- **Format**: Sprite sheet 48x24px (2 frames horizontal)

## Technical Notes
- All debug sprites are **optional** and excluded from production builds
- Camera viewport itself is rendered as a clipping rectangle, not a sprite
- World-to-screen coordinate transformation is mathematical, requires no visual assets
