# PRD — 实现摄像机跟随系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 玩家，I want to 摄像机能够平滑跟随角色移动并正确显示所有游戏元素，So that 我能获得流畅的横向卷轴游戏体验，不会出现画面抖动或元素错位。

## 功能需求
- 摄像机跟随：摄像机 X 轴跟随玩家位置，保持玩家在视口中心偏左 1/3 处
- 边界限制：摄像机不超出关卡左右边界（左边界 X=0，右边界 X=关卡宽度-视口宽度）
- 坐标转换系统：所有游戏对象（玩家、敌人、平台、金币、尖刺、旗帜）的世界坐标正确转换为屏幕坐标进行渲染
- 平滑移动：摄像机位置变化使用线性插值，避免瞬移和抖动
- 视口管理：Canvas 视口尺寸为 800x600px，摄像机视野范围对应世界坐标区域

## 验收标准
- [ ] 玩家向右移动时，摄像机 X 坐标在 100ms 内开始跟随，保持玩家在屏幕 X=266px（视口宽度 1/3）位置
- [ ] 玩家到达关卡左边界（X=0）时，摄像机 X=0 不再左移
- [ ] 玩家到达关卡右边界时，摄像机 X=关卡宽度-800，右侧边界对齐视口右边缘
- [ ] 所有平台、金币、敌人在摄像机移动时保持相对世界位置不变，屏幕坐标=世界坐标-摄像机偏移量
- [ ] 玩家在屏幕中央跳跃时，Y 轴坐标变化正确显示（摄像机 Y 轴固定或根据玩家 Y 轴有限跟随）
- [ ] 摄像机移动速度 ≤ 玩家移动速度的 1.2 倍，避免延迟感
- [ ] 关卡宽度 > 800px 时，摄像机能完整遍历所有区域，无盲区
- [ ] 关卡宽度 ≤ 800px 时，摄像机固定在 X=0，显示完整关卡
- [ ] 帧率 ≥ 60fps 时，摄像机跟随无卡顿，坐标转换计算耗时 < 1ms/帧
- [ ] 玩家死亡或通关时，摄像机停止跟随，保持当前位置

## 边界条件（不做的事）
- 不包含：摄像机缩放功能（Zoom In/Out）
- 不包含：摄像机震动效果（受击/爆炸震屏）
- 不包含：多摄像机切换或过场动画
- 暂不支持：Y 轴动态跟随（摄像机 Y 轴固定在关卡中心高度，或仅在玩家跳跃超过阈值时微调）
- 暂不支持：摄像机预判移动（根据玩家速度提前移动）
- 超出范围：触屏设备的双指缩放手势
