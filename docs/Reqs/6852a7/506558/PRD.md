# PRD — 实现摄像机跟随系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 游戏玩家，I want to 体验流畅的横向卷轴视角跟随，So that 能够清晰看到角色移动和关卡内容，不会因为视角固定而失去游戏体验。

## 功能需求
- 摄像机系统：实现 Camera 类管理视口位置和尺寸
- 跟随逻辑：摄像机 X 轴跟随玩家位置，保持玩家在屏幕中央偏左区域（屏幕宽度 1/3 处）
- 边界限制：摄像机不能超出关卡左边界（x=0）和右边界（关卡总宽度）
- 坐标转换：提供 worldToScreen() 方法，将世界坐标转换为屏幕渲染坐标
- 渲染适配：所有游戏对象（玩家、敌人、平台、金币、尖刺、旗帜）的 render() 方法接收 camera 参数并使用转换后的坐标绘制
- 平滑跟随：摄像机移动使用线性插值（lerp），平滑系数 0.1，避免瞬移感

## 验收标准
- [ ] Camera 类包含 x, y, width, height 属性，初始化时 width=canvas.width, height=canvas.height
- [ ] 玩家 X 坐标 < 屏幕宽度 1/3 时，摄像机 X = 0（贴左边界）
- [ ] 玩家 X 坐标 > 关卡宽度 - 屏幕宽度 2/3 时，摄像机 X = 关卡宽度 - 屏幕宽度（贴右边界）
- [ ] 玩家在中间区域时，摄像机 X = 玩家 X - 屏幕宽度 / 3，保持玩家在屏幕左侧 1/3 位置
- [ ] 摄像机 X 坐标变化使用 lerp 平滑过渡，公式：camera.x += (targetX - camera.x) * 0.1
- [ ] worldToScreen(worldX, worldY) 方法返回 { x: worldX - camera.x, y: worldY - camera.y }
- [ ] 所有游戏对象的 render(ctx, camera) 方法调用 camera.worldToScreen() 获取屏幕坐标后再绘制
- [ ] 玩家移动时，背景平台、金币、敌人同步反向移动（视觉上保持相对位置）
- [ ] 摄像机 X 坐标误差 ≤ 1px 时停止插值计算，避免浮点数抖动
- [ ] 关卡宽度 < 屏幕宽度时，摄像机固定在 x=0（不跟随）

## 边界条件（不做的事）
- 不包含：Y 轴摄像机跟随（本工单仅实现横向卷轴，垂直方向固定显示）
- 不包含：摄像机缩放功能（zoom in/out）
- 不包含：摄像机震动效果（受击/爆炸时的屏幕抖动）
- 不包含：多摄像机切换（固定单一跟随摄像机）
- 暂不支持：摄像机预判（根据玩家移动方向提前偏移视野）
- 超出范围：小地图/雷达系统
