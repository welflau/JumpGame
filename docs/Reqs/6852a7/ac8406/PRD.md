# PRD — 开发玩家角色系统和精灵动画

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 玩家，I want to 控制像素风格角色在关卡中移动、跳跃并查看生命值，So that 我可以开始探索游戏世界并与环境交互。

## 功能需求
## 玩家角色类（Player Class）
- 角色状态管理：位置（x, y）、速度（vx, vy）、方向（left/right）、生命值（3 点初始值）、无敌状态（受伤后 1.5s 无敌帧）
- 精灵动画系统：加载用户提供的像素精灵图序列帧，实现行走动画循环播放（8 帧，每帧 100ms）
- 物理系统：重力加速度 0.8 px/frame²，最大下落速度 15 px/frame
- 碰撞盒：矩形碰撞体，宽度 32px，高度 48px，锚点在底部中心

## 键盘输入控制
- A 键/左方向键：向左移动，速度 4 px/frame，播放行走动画（水平翻转）
- D 键/右方向键：向右移动，速度 4 px/frame，播放行走动画
- 空格键：跳跃，初速度 -12 px/frame，仅在地面时生效，跳跃中不可二段跳
- 无按键：播放静止帧（序列帧第 0 帧）

## 生命值系统
- 初始生命值：3 点
- 受伤机制：碰撞敌人/尖刺时扣 1 点生命值，触发 1.5s 无敌帧（角色闪烁效果，每 100ms 切换可见/不可见）
- 死亡判定：生命值 ≤ 0 时触发游戏失败状态
- UI 显示：左上角显示心形图标 × 当前生命值数量（如：❤❤❤）

## 渲染系统
- Canvas 绘制：使用 drawImage 绘制当前动画帧
- 动画帧切换：根据移动状态和时间戳自动切换帧索引
- 方向翻转：使用 Canvas scale(-1, 1) 实现水平镜像

## 验收标准
- [ ] 按下 A 键后，角色以 4 px/frame 速度向左移动，行走动画以 100ms/帧速率循环播放（8 帧），角色精灵水平翻转
- [ ] 按下 D 键后，角色以 4 px/frame 速度向右移动，行走动画以 100ms/帧速率循环播放（8 帧），角色精灵正向显示
- [ ] 在地面时按空格键，角色以 -12 px/frame 初速度向上跳跃，跳跃过程中受重力 0.8 px/frame² 影响，最大下落速度不超过 15 px/frame
- [ ] 跳跃过程中再次按空格键，角色不响应（禁止二段跳）
- [ ] 无按键输入时，角色显示静止帧（序列帧索引 0）
- [ ] 角色碰撞敌人或尖刺后，生命值扣减 1 点，触发 1.5s 无敌帧，期间角色每 100ms 切换可见/不可见状态（闪烁 15 次）
- [ ] 无敌帧期间再次碰撞敌人/尖刺，不扣生命值
- [ ] 生命值 ≤ 0 时，游戏状态切换为「失败」，角色停止响应输入
- [ ] 左上角 UI 显示心形图标数量 = 当前生命值（初始 3 个心形，受伤后实时减少）
- [ ] 角色碰撞盒尺寸为 32px × 48px，锚点在底部中心，与平台边缘对齐时角色不悬空
- [ ] 加载用户提供的精灵图后，行走动画帧数 = 8 帧，每帧尺寸 64px × 64px（假设），图片路径错误时 console 输出错误日志
- [ ] 角色移动时帧率保持 ≥ 60fps，动画播放流畅无卡顿

## 边界条件（不做的事）
## 不做的事
- 不实现攻击系统（本工单仅负责移动和受伤）
- 不实现金币收集逻辑（由后续工单负责）
- 不实现摄像机跟随（由后续工单负责）
- 不实现关卡地图和平台（本工单仅实现角色类，测试时使用临时平台）
- 不实现敌人 AI（由后续工单负责）
- 不实现通关/失败 UI 界面（本工单仅切换游戏状态，UI 由后续工单负责）
- 不实现音效系统（由后续工单负责）

## 暂不支持
- 手柄/触摸屏输入（仅支持键盘）
- 可配置的按键映射（按键硬编码为 A/D/空格）
- 动态调整重力/跳跃参数（数值硬编码）

## 超出范围
- 多角色切换
- 角色皮肤系统
- 技能系统
- 等级/经验系统

## 资产需求线索
## 必需资产
- **角色精灵图**（用户已提供）：像素风格行走动画序列帧，PNG 格式，透明背景，包含 8 帧行走动画，单帧尺寸建议 64px × 64px
- **心形图标**：像素风格红色心形，尺寸 24px × 24px，用于 UI 生命值显示

## 可选资产（后续工单）
- 受伤音效：短促的「嘶」声，时长 0.3s
- 跳跃音效：轻快的「啵」声，时长 0.2s
- 死亡音效：低沉的「咚」声，时长 0.5s

## 临时测试资产（开发阶段）
- 临时平台：纯色矩形（灰色 #808080），用于测试角色站立和跳跃
- 临时背景：纯色画布（天蓝色 #87CEEB）

## 资产加载要求
- 精灵图路径：`assets/sprites/player_walk.png`（相对路径）
- 心形图标路径：`assets/ui/heart.png`
- 加载失败时显示红色占位矩形 + console 错误日志
