# PRD — 开发UI系统和游戏反馈

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 玩家，I want to 在游戏中看到清晰的生命值和金币数量显示，并在通关或失败时获得明确的反馈，So that 我能实时了解游戏状态并决定是否重新开始游戏。

## 功能需求
## HUD 显示系统
- 生命值显示：使用心形图标表示剩余生命值（最多 3 颗心）
- 金币计数器：显示当前收集的金币数量 / 总金币数（格式：X/20）
- HUD 固定在屏幕左上角，不随摄像机滚动

## 游戏结束界面
- 通关界面：显示「Victory!」文字 + 收集的金币数量 + 重新开始按钮
- 失败界面：显示「Game Over」文字 + 重新开始按钮
- 界面背景半透明黑色遮罩（alpha=0.7）

## 视觉反馈系统
- 受伤反馈：玩家受到伤害时角色精灵闪烁红色 3 次（每次 100ms），总计 300ms
- 收集反馈：拾取金币时播放缩放动画（金币从原始大小放大 1.2 倍后消失，持续 200ms）
- 死亡反馈：生命值归零时角色精灵淡出（alpha 从 1.0 降至 0，持续 500ms）后显示失败界面

## 重新开始功能
- 点击重新开始按钮后 100ms 内重置游戏状态（玩家位置、生命值、金币计数、敌人位置）
- 重置后立即关闭结束界面，恢复游戏进行状态

## 验收标准
- [ ] HUD 生命值显示：初始显示 3 颗完整心形图标，每次受伤减少 1 颗，生命值归零时心形全部消失
- [ ] HUD 金币计数：初始显示「0/20」，每拾取 1 枚金币数字 +1，格式始终为「当前数/20」
- [ ] HUD 位置固定：HUD 元素距离屏幕左上角固定偏移（生命值：左 20px 上 20px，金币计数：左 20px 上 60px），摄像机滚动时 HUD 不移动
- [ ] 通关界面触发：玩家碰撞旗帜后 200ms 内显示通关界面，界面包含「Victory!」文字（字号 48px）+ 金币统计文字（格式：「Coins: X/20」）+ 重新开始按钮
- [ ] 失败界面触发：生命值 ≤ 0 时，角色淡出动画完成后（500ms）显示失败界面，界面包含「Game Over」文字（字号 48px）+ 重新开始按钮
- [ ] 界面遮罩效果：通关/失败界面背景为全屏黑色矩形，透明度 alpha=0.7
- [ ] 受伤闪烁效果：玩家受伤时角色精灵颜色变为红色（RGB: 255,100,100），闪烁 3 次（显示 100ms → 隐藏 100ms 为 1 次循环），总计 600ms 后恢复正常颜色
- [ ] 金币收集动画：金币被拾取时播放缩放动画（scale 从 1.0 → 1.2 → 0，持续 200ms），动画结束后金币对象从场景移除
- [ ] 死亡淡出动画：生命值归零时角色精灵 alpha 从 1.0 线性降至 0，持续 500ms
- [ ] 重新开始功能：点击重新开始按钮后 100ms 内完成以下重置：玩家位置回到起点（x=50, y=ground-100）、生命值=3、金币计数=0、所有金币对象重新生成、所有敌人回到初始位置
- [ ] 按钮交互反馈：鼠标悬停重新开始按钮时按钮颜色变亮 20%，点击时缩放至 0.95 倍（持续 100ms）后恢复
- [ ] 字体渲染：所有文字使用像素风格字体（推荐 "Press Start 2P" 或等宽像素字体），抗锯齿关闭以保持像素风格一致性

## 边界条件（不做的事）
## 不包含的功能
- 不实现音效系统（受伤音效、收集音效、背景音乐等）
- 不实现暂停菜单功能
- 不实现设置界面（音量调节、按键映射等）
- 不实现多关卡选择界面
- 不实现分数系统（仅统计金币数量，不计算时间奖励或连击奖励）
- 不实现排行榜或成就系统
- 不实现存档功能（每次重新开始都是完全重置）

## 暂不支持的平台特性
- 不适配移动端触摸操作（仅支持键盘操作）
- 不支持手柄输入
- 不支持屏幕尺寸自适应（固定 Canvas 尺寸）

## 超出当前工单范围
- 玩家移动/跳跃逻辑（已在前序工单完成）
- 敌人 AI 行为（已在前序工单完成）
- 关卡地图布局（已在前序工单完成）
- 碰撞检测系统（已在前序工单完成）
- 摄像机跟随逻辑（已在前序工单完成）

## 资产需求线索
## 图标资产
- 心形图标（生命值显示）：16x16px 像素风格，需要完整心形和空心形两种状态
- 金币图标（HUD 计数器前缀）：12x12px 像素风格，与场景中金币视觉一致

## UI 元素
- 重新开始按钮：120x40px 矩形按钮，像素风格边框，需要正常/悬停/按下三种状态的视觉设计
- 半透明遮罩：纯色填充即可，无需额外资产

## 字体资产
- 像素风格字体文件：推荐使用开源字体 "Press Start 2P"（Google Fonts）或 "Silkscreen"，需支持英文字符和数字

## 特效资产（可选，可用代码实现）
- 受伤闪烁效果：通过代码修改精灵颜色实现，无需额外资产
- 金币收集动画：通过代码缩放实现，无需额外资产
- 死亡淡出效果：通过代码修改 alpha 实现，无需额外资产

## 复用现有资产
- 玩家角色精灵图（已提供）
- 场景金币对象（已在关卡设计中实现）
- 旗帜对象（已在关卡设计中实现）
