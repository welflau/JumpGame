# PRD — 开发UI系统和游戏反馈

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 玩家，I want to 在游戏中看到清晰的生命值和金币数量，并在通关或失败时得到明确反馈，So that 我能实时了解游戏状态并决定是否重新挑战。

## 功能需求
## HUD 显示系统
- 生命值显示：左上角显示 3 个心形图标，每个代表 1 点生命值
- 金币计数器：右上角显示「🪙 x [数量]」格式，实时更新收集数量
- UI 层级：HUD 固定在屏幕坐标，不受摄像机滚动影响

## 游戏状态提示界面
- 通关界面：触碰旗帜后显示半透明遮罩 + 「Victory!」文字 + 金币统计「Coins: [X]/20」
- 失败界面：生命值归零后显示「Game Over」文字 + 当前金币数
- 重新开始按钮：两种界面均显示「Restart」按钮，点击后重置关卡

## 视觉反馈系统
- 受伤反馈：玩家受到伤害时，角色精灵闪烁红色 3 次（每次 100ms），同时心形图标消失动画
- 收集反馈：拾取金币时播放缩放消失动画（200ms），金币计数器数字跳动放大效果
- 死亡反馈：生命值归零时角色透明度渐变至 0（500ms）后显示失败界面
- 通关反馈：触碰旗帜时旗帜播放挥动动画，摄像机停止跟随

## 验收标准
- [ ] 生命值心形图标在屏幕左上角固定显示，初始 3 个，每次受伤减少 1 个，图标消失伴随 150ms 淡出动画
- [ ] 金币计数器在屏幕右上角显示「🪙 x 0」，拾取金币后 100ms 内数字更新并播放 1.2 倍放大→还原动画（持续 300ms）
- [ ] 玩家受到伤害时，角色精灵以 100ms 间隔闪烁红色（#FF0000 叠加）共 3 次，总时长 300ms
- [ ] 拾取金币时，金币精灵播放缩放动画（1.0→1.5→0 倍，200ms）后从画面移除
- [ ] 生命值 ≤ 0 时，500ms 内角色透明度从 1.0 渐变至 0，随后显示失败界面
- [ ] 失败界面包含：半透明黑色遮罩（opacity: 0.7）+ 「Game Over」文字（48px 红色）+ 「Coins: [X]/20」（24px 白色）+ 「Restart」按钮（120x40px 绿色）
- [ ] 触碰旗帜后 200ms 内显示通关界面，包含「Victory!」文字（64px 金色）+ 金币统计 + 「Restart」按钮
- [ ] 点击「Restart」按钮后 100ms 内重置：玩家位置回到起点、生命值恢复 3 点、金币计数归零、所有金币重新生成、敌人回到初始位置
- [ ] HUD 元素使用屏幕坐标系，摄像机滚动时保持固定位置不移动
- [ ] 通关/失败界面显示时，玩家输入被禁用（无法移动/跳跃），仅「Restart」按钮可交互
- [ ] 心形图标尺寸 32x32px，间距 8px，使用像素风格红色心形
- [ ] 金币图标尺寸 24x24px，计数文字使用像素字体 20px 白色描边

## 边界条件（不做的事）
## 不做的功能
- 不实现暂停菜单（本工单仅处理通关/失败状态）
- 不添加音效系统（音效由后续工单处理）
- 不实现关卡选择界面（单关卡游戏）
- 不支持触摸/手柄输入（仅键盘操作）
- 不实现分数系统（仅统计金币数量）
- 不添加计时器功能
- 不实现多语言切换（UI 文字固定英文）

## 暂不支持
- 自定义 UI 皮肤/主题
- 动态难度调整
- 成就系统
- 排行榜

## 技术边界
- UI 渲染使用 Canvas 2D API，不引入 UI 框架
- 不使用 WebGL/Pixi.js 等渲染库
- 不实现 UI 动画缓动库（使用线性插值）

## 资产需求线索
## 图标资产
- 心形图标（heart.png）：32x32px 像素风格，红色填充，需要完整和空心两种状态
- 金币图标（coin.png）：24x24px 像素风格，金黄色圆形

## UI 元素
- 像素字体文件（pixel-font.ttf 或使用 Canvas 默认等宽字体模拟）
- 按钮背景：120x40px 圆角矩形，绿色填充（#4CAF50），悬停时变为 #45A049

## 特效资产
- 收集粒子（可选）：金色星星/光点精灵，8x8px，用于金币收集时的粒子效果
- 受伤特效（可选）：红色闪光叠加层或冲击波精灵

## 背景元素
- 半透明遮罩：纯色黑色，通过 Canvas fillStyle rgba(0,0,0,0.7) 实现，无需独立资产

## 文字样式参考
- 标题文字：像素风格，描边效果（白色文字 + 2px 黑色描边）
- 计数文字：等宽字体，便于数字对齐

**[假设]** 如果没有现成像素字体文件，使用 Canvas font = '20px monospace' 并手动绘制描边效果（strokeText + fillText）来模拟像素风格。
