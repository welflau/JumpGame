# PRD — 实现物理引擎和碰撞检测系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 游戏开发者，I want to 实现完整的物理引擎和碰撞检测系统，So that 玩家角色和敌人能够与平台、道具、陷阱进行真实的物理交互，为游戏提供流畅的移动和跳跃体验。

## 功能需求
- 重力系统：所有实体受恒定向下重力影响（GRAVITY = 0.8 px/frame²）
- 跳跃机制：玩家按空格键施加向上初速度（JUMP_VELOCITY = -15 px/frame），支持单次跳跃（落地后重置）
- AABB 碰撞检测：实现轴对齐包围盒算法，检测矩形实体间的重叠
- 碰撞响应：实体与平台碰撞时停止下落，速度归零，位置修正到平台表面
- Entity 基类：包含 position(x, y)、velocity(vx, vy)、size(width, height)、grounded(布尔值) 属性
- 碰撞分组：区分 solid 平台（阻挡）、trigger 触发器（金币/旗帜/尖刺，不阻挡但触发事件）
- 边界检测：实体超出关卡左右边界时限制移动，超出下边界判定死亡

## 验收标准
- [ ] 玩家站在平台上时 grounded=true，velocity.y=0，离开平台后 grounded=false 且每帧 velocity.y += GRAVITY
- [ ] 按下空格键时，若 grounded=true 则 velocity.y = -15，角色向上移动，grounded 变为 false
- [ ] 玩家从高处落下碰到平台时，position.y 修正到平台顶部（player.y + player.height = platform.y），velocity.y 归零
- [ ] AABB 检测函数返回布尔值：两矩形重叠时返回 true，分离时返回 false（测试用例：重叠矩形 [0,0,10,10] 与 [5,5,10,10] 返回 true，分离矩形 [0,0,10,10] 与 [20,20,10,10] 返回 false）
- [ ] 玩家水平移动速度 = 5 px/frame，碰到关卡左边界（x=0）或右边界（x=levelWidth）时停止移动
- [ ] 玩家 y 坐标 > levelHeight + 100 时触发死亡事件（生命值清零）
- [ ] 敌人实体继承 Entity 基类，自动应用重力和碰撞检测（巡逻兵在平台上移动不会掉落）
- [ ] 金币/旗帜/尖刺设置为 trigger 类型，玩家碰撞时触发事件但不阻挡移动（玩家可穿过金币继续移动）
- [ ] 碰撞检测每帧执行时间 < 5ms（60fps 下单帧预算 16.67ms，碰撞检测占用 < 30%）
- [ ] Entity 基类包含 update() 方法，自动更新位置（position.x += velocity.x, position.y += velocity.y）和应用重力

## 边界条件（不做的事）
- 不包含：斜坡/曲面碰撞（仅支持矩形 AABB），旋转碰撞体，物理材质（摩擦力/弹性系数）
- 暂不支持：二段跳/空中冲刺，可破坏平台，移动平台（动态碰撞体）
- 超出范围：完整物理引擎特性（刚体动力学/关节约束/流体模拟），像素级精确碰撞（使用包围盒简化）
- [假设] 重力加速度和跳跃初速度为固定数值，不支持运行时调整
- [假设] 碰撞检测仅在主循环每帧执行一次，不支持子步长模拟
- [假设] 所有实体碰撞盒与精灵图尺寸一致，不支持自定义偏移
