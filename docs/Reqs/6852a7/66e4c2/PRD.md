# PRD — 实现物理引擎和碰撞检测系统

> 所属需求：横向卷轴平台跳跃游戏开发

## 用户故事
As a 游戏开发者，I want to 实现完整的物理引擎和碰撞检测系统，So that 玩家角色、敌人和场景元素能够正确地进行物理交互，为游戏提供真实的跳跃、移动和碰撞反馈。

## 功能需求
## 核心功能

### 1. 重力系统
- 所有实体受重力影响，垂直速度每帧增加固定加速度值
- 重力加速度：0.8 px/frame²
- 最大下落速度限制：15 px/frame
- 实体落地后垂直速度归零

### 2. AABB 碰撞检测
- 实现轴对齐包围盒（Axis-Aligned Bounding Box）算法
- 检测矩形碰撞盒之间的重叠
- 返回碰撞方向（上/下/左/右）和重叠深度
- 支持实体与平台、实体与实体的碰撞检测

### 3. 物理响应机制
- 碰撞后位置修正：将实体推出碰撞区域
- 速度修正：碰撞方向的速度分量归零
- 落地判定：检测实体底部与平台顶部的碰撞
- 头顶碰撞：跳跃时撞到平台底部，向下速度归零

### 4. Entity 基类设计
- 位置属性：x, y（像素坐标）
- 速度属性：vx, vy（px/frame）
- 碰撞盒属性：width, height, offsetX, offsetY
- 物理状态：isOnGround（是否在地面）
- 方法：update(), applyGravity(), checkCollision(), resolveCollision()

### 5. 平台碰撞系统
- 支持单向平台（只能从下方穿过，从上方落下时站立）
- 支持双向平台（四个方向都阻挡）
- 平台边缘检测：实体完全离开平台边缘后失去支撑

### 6. 跳跃机制物理基础
- 跳跃初速度：-12 px/frame（向上为负）
- 只能在 isOnGround=true 时跳跃
- 跳跃时立即设置 isOnGround=false
- 变高跳跃支持：按住跳跃键时间越长，跳得越高（最多持续 10 帧）

## 验收标准
- [ ] 实体从空中下落时，每帧垂直速度增加 0.8 px/frame，最大下落速度不超过 15 px/frame
- [ ] 实体落在平台上后，isOnGround 标志设为 true，垂直速度归零，位置精确对齐平台顶部（误差 ≤ 1px）
- [ ] AABB 碰撞检测函数在两个矩形重叠时返回 true，不重叠时返回 false，准确率 100%
- [ ] 碰撞方向判定准确：从上方落下判定为 'bottom'，从下方跳起判定为 'top'，从左右判定为 'left'/'right'
- [ ] 实体与平台发生碰撞后，位置修正误差 ≤ 1px，不会出现嵌入或抖动现象
- [ ] 跳跃时给予初速度 -12 px/frame，按住跳跃键 10 帧内持续减少下落加速度，实现变高跳跃
- [ ] 实体头顶撞到平台底部时，向上速度立即归零，不会穿透平台
- [ ] 单向平台从下方穿过时不触发碰撞，从上方落下时正常站立（检测实体底部 y 坐标 ≤ 平台顶部 y 坐标）
- [ ] Entity 基类包含以下属性：x, y, vx, vy, width, height, offsetX, offsetY, isOnGround
- [ ] Entity 基类包含以下方法：update(), applyGravity(), getCollisionBox(), checkCollisionWith(other)
- [ ] 实体完全离开平台边缘（碰撞盒右边缘 < 平台左边缘 或 碰撞盒左边缘 > 平台右边缘）后，isOnGround 设为 false
- [ ] 多个实体同时与同一平台碰撞时，每个实体独立处理碰撞响应，不会相互干扰
- [ ] 物理系统帧率稳定在 60 FPS 时，跳跃高度误差 ≤ 5%，落地时间误差 ≤ 3 帧
- [ ] 碰撞检测性能：100 个实体同时运行时，每帧碰撞检测耗时 ≤ 5ms（Chrome DevTools Performance 测量）

## 边界条件（不做的事）
## 不做的事

- **不实现**：旋转碰撞盒（所有碰撞盒保持轴对齐）
- **不实现**：圆形/多边形碰撞检测（仅支持矩形 AABB）
- **不实现**：物理材质系统（弹性、摩擦力、阻尼等）
- **不实现**：刚体动力学（角速度、力矩、质量等）
- **不实现**：连续碰撞检测（CCD，防止高速物体穿透）
- **不实现**：空间分区优化（四叉树、网格等，当前实体数量 < 100 无需优化）
- **不实现**：斜坡/曲面碰撞（仅支持水平和垂直平台）
- **不包含**：具体的玩家控制逻辑（移动、跳跃输入处理由后续工单实现）
- **不包含**：敌人 AI 逻辑（仅提供物理基础，AI 由后续工单实现）
- **不包含**：伤害判定和生命值系统（由后续战斗系统工单实现）
- **不包含**：视觉反馈（碰撞特效、音效等由后续工单实现）
- **暂不支持**：移动平台（电梯、传送带等动态平台）
- **暂不支持**：水/风等环境力场
- **超出范围**：网络同步的物理预测和回滚
- **超出范围**：物理引擎编辑器/调试工具 UI
